# Olist 유의판매자 조기 탐지 시스템

> 배송 지연의 90%는 판매자 처리 지연에서 발생한다.
> ML 기반 조기 경보 시스템으로 연간 **$1.05M** CS 비용 절감을 추정했다.

## 프로젝트 개요

브라질 최대 이커머스 마켓플레이스 Olist의 실제 주문 데이터(2016-2018, ~10만 건)를 분석하여
**플랫폼 품질을 저해하는 "유의판매자(Seller of Note)"를 조기에 식별**하고,
운영팀이 즉시 활용 가능한 **대시보드 + 자동 메일링 시스템**을 구축했다.

### 문제 정의

| 문제 | 수치 |
|------|------|
| 배송 지연 시 리뷰 점수 | 4.13점 → **2.55점** (1.6점 하락) |
| 블랙프라이데이 부정 리뷰 비율 | 14.4% → **21.7%** |
| 나쁜 리뷰 고객 이탈률 | **80%** |
| 나쁜 리뷰 1건의 추정 피해 비용 | CS + 보상 + 이탈 = **$95/건** |

### 해결 방안

```
EDA 인사이트                 ML 모델                    운영 시스템
─────────────────    ─────────────────────    ─────────────────────
지연 90%가 판매자    →  RandomForest 기반       →  Streamlit 대시보드
처리 단계에서 발생       유의판매자 분류 모델        자동 메일링 (4단계)
                        10-Fold CV 검증             일별 리스크 리포트
```

## 핵심 발견

### 1. 배송 지연의 원인은 물류가 아니라 판매자다

| 배송 단계 | 동일 주 | 다른 주 | 차이 |
|-----------|---------|---------|------|
| **판매자 처리 시간** | 2.3일 | 2.4일 | 0.1일 (거의 없음) |
| **물류 배송 시간** | 4.3일 | 11.3일 | 7.0일 |

판매자 처리 시간은 지역과 무관하게 **약 2일로 일정**하다.
물류 지연은 인프라 문제이지만, **판매자 처리 지연은 관리 가능한 변수**다.

### 2. 비율 기반 지표가 평균보다 효과적이다

| 변수 조합 | Spearman 상관계수 |
|-----------|-------------------|
| 평균 지연일수 vs 부정 리뷰율 | 0.051 (거의 없음) |
| **지연 비율** vs 부정 리뷰율 | **0.253** (뚜렷함) |

→ "얼마나 오래 늦었는가"보다 **"얼마나 자주 늦는가"**가 고객 불만족과 직결된다.

### 3. 카테고리별 구조적 지연 패턴이 존재한다

| 카테고리 | 판매자 처리 | 전체 배송 | 느린 판매자 비율 |
|----------|------------|----------|-----------------|
| office_furniture | **9.7일** | 20.4일 | 29.0% |
| baby | 3.1일 | 14.2일 | 8.9% |
| health_beauty | 1.8일 | 10.8일 | 2.1% |

ANOVA 검정 결과 카테고리 간 차이 **통계적으로 유의** (F=160.5, p<0.001).

```

                           ↓
┌─────────────────────────────────────────────────────────────┐
│  02-04. 탐색적 데이터 분석 (EDA)                              │

                           ↓
┌─────────────────────────────────────────────────────────────┐
│  05-06. 피처 엔지니어링 & 검증                                │
│  27개 파생변수 생성 → VIF 분석(전부 <5) → ANOVA 검증           │

┌─────────────────────────────────────────────────────────────┐
│  07. 머신러닝 모델링                                          │

└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│  08. 비용 분석 + 운영 시스템                                   │

└─────────────────────────────────────────────────────────────┘
```

## 유의판매자 정의 및 모델링

### 정의 기준

3개 지표가 **모두** 그룹별 임계값을 초과하면 유의판매자로 분류한다.

| 지표 | 의미 | 가중치 |
|------|------|--------|
| `processing_delay_rate` | 카테고리 평균보다 처리 느린 주문 비율 | 15% |
| `seller_delay_rate` | 출고 기한 초과 주문 비율 | **50%** |
| `negative_review_rate` | 불만족 리뷰(1-3점) 비율 | 35% |

### 차등 임계값

| 판매자 그룹 | 인원 | 기준 | 유의판매자 | 비율 |
|-------------|------|------|-----------|------|
| 상위 25% (대형) | 663명 | 75% 분위수 (엄격) | 109명 | 16.4% |
| 중간 25-50% | 737명 | 90% 분위수 (관대) | 7명 | 0.9% |
| **합계** | **1,400명** | - | **116명** | **8.3%** |

→ 판매량이 많은 대형 판매자에게 **더 엄격한 기준** 적용.

### 4단계 리스크 등급

| 등급 | 확률 범위 | 조치 |
|------|----------|------|
| RED | >= 0.8 | 경고 메일, 2주 개선 기한, 신규 등록 동결 |
| ORANGE | 0.4 - 0.79 | 월간 성과 리포트, 교육 프로그램 |
| YELLOW | 0.3 - 0.39 | 분기별 재평가 |
| GREEN | < 0.3 | 인센티브 부여, 우수 사례 공유 |

## 실행 방법

### 환경 설정

```bash
git clone https://github.com/YuSeok-Kwon/Olist-Predictive-Modeling.git
cd Olist-Predictive-Modeling
pip install -r requirements.txt
```

### 대시보드 실행

```bash
streamlit run src/dashboard.py
```

실행 후 `http://localhost:8501` 접속.
사이드바의 **"위험 판매자 데이터 갱신"** 버튼으로 ML 파이프라인 원클릭 실행.

```bash
# 전처리부터 순차 실행
jupyter notebook notebooks/
```
